# Week 07 任务总结报告

## 1. GradientDescentOLS 实现说明

### 核心实现要点：
- **梯度计算**：使用 MSE loss 的导数 ∇L = (2/n) X^T (Xθ - y)
- **参数更新**：θ ← θ - η∇L
- **收敛判断**：连续两次迭代的 loss 差异小于 tol 时停止
- **Mini-batch 支持**：每次随机抽取 batch_fraction 比例的样本
- **随机种子固定**：确保结果可复现

### 截距项处理：
- 在特征标准化**之后**手动添加截距列（全1列）
- 确保截距列不被标准化，保持其物理意义

## 2. Full-Batch vs Mini-Batch 学习曲线差异

### Full-Batch GD：
- **特点**：每次使用全部数据计算梯度
- **曲线特征**：平滑下降，无震荡
- **收敛速度**：稳定但每轮计算量大

### Mini-Batch GD：
- **特点**：每次使用部分数据（20%）
- **曲线特征**：震荡下降，噪声较大
- **收敛速度**：每轮更快，但整体可能需要更多轮

## 3. 最佳学习率选择

经过 5 个学习率的验证集评估，最佳学习率为：**0.1**

## 5. 防止数据泄露的标准化策略

### 正确流程：

```
# 1. 先划分数据
X_train, X_temp, y_train, y_temp = train_test_split(...)

# 2. 只用训练集拟合 scaler
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)

# 3. 用训练集的统计量转换验证集和测试集
X_val_scaled = scaler.transform(X_val)
X_test_scaled = scaler.transform(X_test)
```

### 为什么要这样做？
- **防止信息泄露**：如果先用全数据拟合 scaler，验证/测试集会'看到'训练集的信息
- **模拟真实场景**：部署时只能使用训练时的统计量
- **避免过于乐观**：数据泄露会导致泛化能力被高估

### 关键点：
- ✅ 只标准化特征列
- ✅ 截距列在标准化后添加，不参与标准化
- ✅ 验证/测试集只使用 transform，不使用 fit_transform
